## Project Overview

This is a **Shopify Dawn theme (v15.4.1)** with custom section additions. It follows the standard Shopify Online Store 2.0 theme architecture using Liquid templating, JSON templates, and section-based rendering.

## Development Commands

```bash
# Serve theme locally (requires Shopify CLI and a connected store)
shopify theme dev

# Push theme to a development store
shopify theme push

# Pull latest theme from store
shopify theme pull

# Check Liquid for errors
shopify theme check
```

## Architecture

- **layout/** — Base layout wrappers (`theme.liquid`, `password.liquid`). All pages render through `theme.liquid` which loads global JS (constants, pubsub, global, details-disclosure, details-modal, search-form) and base CSS.
- **templates/** — JSON templates that compose sections. Each page type (product, collection, cart, etc.) has a `.json` file defining which sections appear and in what order.
- **sections/** — Liquid section files, each self-contained with markup, schema, and stylesheet references. Sections are the primary unit of work.
- **snippets/** — Reusable Liquid partials rendered via `{% render %}`.
- **assets/** — CSS and JS files. CSS follows a `component-*.css` naming convention for per-component styles and `section-*.css` for section-specific styles. JS files are loaded with `defer` and use Web Components (custom elements) extensively.
- **config/** — `settings_schema.json` defines theme-level settings; `settings_data.json` stores current values (auto-generated, avoid manual edits).
- **locales/** — Translation files. `en.default.json` is the source of truth; schema translations use `en.default.schema.json`.

## Custom Sections (non-standard Dawn)

These sections were added on top of the base Dawn theme and are the most likely areas of active development:

- `animated-featured-product.liquid` — Trending/featured product with animation
- `banner-with-product.liquid` — Full-width banner with overlapping product cards
- `insta-feed.liquid` — Instagram-style feed with hover overlays (image/video blocks)
- `marquee-banner.liquid` — Dual-mode banner (marquee heading or sliding gallery)
- `our-story-split.liquid` — 3-column layout with scroll-animated image and marquee ticker
- `review-section.liquid` — Reviews slider using Swiper.js
- `section-collection-tabs.liquid` — Tabbed collection product grid
- `shop-the-look.liquid` — Hotspot-based product panel with mobile slider
- `category.liquid` — Category page section

Supporting custom snippets: `collection-tabs-card.liquid`, `icon-with-text.liquid`, `icon-accordion.liquid`.

## Key Patterns

- **Web Components**: JS in `assets/` defines custom elements (e.g., `cart-drawer.js`, `product-form.js`). Extend `HTMLElement` and register with `customElements.define()`.
- **Pub/Sub**: `pubsub.js` provides a global event bus used for cross-component communication (cart updates, variant changes).
- **Section Rendering API**: Sections use `{{ section.id }}` for unique scoping and support Shopify's section rendering API for dynamic updates without full page reload.
- **CSS Loading**: Component CSS is loaded on-demand via `{% stylesheet %}` tags or `<link>` in sections that need them, keeping unused CSS off pages that don't use the component.
- **Translations**: Use `{{ 'key' | t }}` in Liquid and `"t:key"` references in schema JSON. All user-facing strings should be translatable.

## Things to Watch

- `config/settings_data.json` is generated by the theme editor — do not manually edit or merge this file.
- Custom sections may include inline `<style>` and `<script>` blocks with section-scoped CSS using `#section-{{ section.id }}` selectors.
- Swiper.js is loaded via CDN in `review-section.liquid` — not bundled locally.
